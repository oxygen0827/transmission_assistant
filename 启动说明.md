# 文件传输助手 — 本地开发启动说明

## 目录结构

```
file-assistant/
├── backend/        后端 (FastAPI)
├── frontend/       前端 (Vue3 + Vite)
├── data/           文件存储目录（自动生成）
├── .env            API 密钥配置（首次需手动创建）
└── 启动说明.md     本文件
```

---

## 第一次使用：配置 .env

在 `file-assistant/` 目录下创建 `.env` 文件，内容如下：

```
GLM_API_KEY=你的智谱API密钥
GLM_MODEL=glm-4-flash
GLM_VISION_MODEL=glm-4v-flash
```

> 智谱 API Key 在 https://open.bigmodel.cn/ 注册后获取。

---

## 启动后端

打开 **PowerShell**，执行：

```powershell
cd "C:\Users\19051\Desktop\ai_deploy\transmission asistor\file-assistant"
$env:PYTHONPATH = $PWD
uvicorn backend.main:app --reload --port 8000
```

看到以下输出表示启动成功：

```
INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
```

> 后端运行期间**不要关闭**这个窗口。

---

## 启动前端

另开一个 **PowerShell** 窗口，执行：

```powershell
cd "C:\Users\19051\Desktop\ai_deploy\transmission asistor\file-assistant\frontend"
npm run dev
```

看到以下输出表示启动成功：

```
  ➜  Local:   http://localhost:5173/
```

打开浏览器访问 **http://localhost:5173** 即可使用。

---

## 每次测试流程

1. 启动后端（PowerShell 窗口 1）
2. 启动前端（PowerShell 窗口 2）
3. 浏览器打开 http://localhost:5173

> 前后端需要**同时运行**，缺一不可。

---

## 常见问题

| 现象 | 原因 | 解决 |
|------|------|------|
| 页面转圈 / 上传失败 | 后端未启动 | 按上方步骤启动后端 |
| AI 分析全部失败 | `.env` 中 `GLM_API_KEY` 为空 | 填入正确的 API Key |
| `npm run dev` 报错 | 依赖未安装 | 在 `frontend/` 目录执行 `npm install` |
| `ModuleNotFoundError` | PYTHONPATH 未设置 | 确保执行了 `$env:PYTHONPATH = $PWD` |
| 后端依赖缺失 | requirements 未安装 | 执行 `pip install -r requirements.txt` |

---

## 停止服务

在对应 PowerShell 窗口按 `Ctrl + C` 即可停止。
